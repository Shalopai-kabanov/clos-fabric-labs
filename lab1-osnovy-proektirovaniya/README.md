6. Архитектура сети
Underlay:
L3 сеть
point-to-point линki (/31)
протокол маршрутизации - eBGP
Spine:
транзитный уровень
не участвует в L2
Leaf:
точка подключения хостов
выполняет функции L3 gateway
Access:
подключение конечных устройств
на Leaf3 два хоста в одной подсети

7. План работ
Развертывание топологии в эмуляторе
Базовая настройка устройств (hostname, ip routing)
Настройка loopback-интерфейсов
Настройка underlay линков (/31)
Проверка L3 связности между соседями
Настройка eBGP между spine и leaf
Анонс loopback-адресов
Проверка таблиц маршрутизации
Настройка access-портов на leaf-коммутаторах
Назначение IP-адресов хостам
Настройка шлюзов на leaf
Проверка связности между хостами

8. Ожидаемый результат
Все BGP-сессии установлены
Loopback-адреса доступны между всеми устройствами
Хосты имеют IP-связность:
внутри своей подсети
между разными leaf через fabric
Трафик проходит через spine-уровень
