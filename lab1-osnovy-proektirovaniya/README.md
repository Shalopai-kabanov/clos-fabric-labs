План работ по построению CLOS IP Fabric
1. Цель работ
Развернуть CLOS-топологию (2 spine, 3 leaf), настроить underlay-сеть с использованием L3 point-to-point линков и обеспечить IP-связность между всеми узлами.

2. Состав стенда
Устройства:

Spine1
Spine2
Leaf1
Leaf2
Leaf3

Тип топологии:

CLOS (spine-leaf)
Полносвязная схема между spine и leaf

3. Адресное пространство

3.1 Underlay сеть
Сеть: 10.0.0.0/16
Тип линков: point-to-point (/31)
3.2 Loopback адресация
Сеть: 10.255.0.0/24
Назначение:
router-id
BGP peering (в дальнейшем)
VTEP (в дальнейшем)

4. Этапы выполнения работ
4.1 Подготовка инфраструктуры
Развернуть виртуальные устройства
Назначить имена (hostname)
Проверить соответствие интерфейсов схеме

4.2 Базовая настройка устройств
На всех узлах:

включить IP-маршрутизацию
отключить лишние сервисы

4.3 Настройка loopback интерфейсов
На каждом устройстве:

создать Loopback0
назначить /32 адрес согласно плану

4.4 Настройка underlay линков
Настроить IP-адресацию на всех физических интерфейсах
Использовать /31 для каждого линка
Включить интерфейсы (no shutdown)

4.5 Проверка L3-связности
Проверить доступность соседей по каждому point-to-point линку
Убедиться в корректности IP-адресации

4.6 Настройка динамической маршрутизации (BGP underlay)
Назначить автономные системы:
Spine - AS 65000
Leaf - индивидуальные AS
Настроить eBGP-соседства между spine и leaf
Анонсировать loopback-адреса

4.7 Проверка маршрутизации
Проверить состояние BGP-сессий
Убедиться, что все loopback-адреса доступны
Проверить таблицу маршрутизации

4.8 Сквозная проверка связности
Проверить доступность между всеми leaf-узлами
Проверить прохождение трафика через spine

5. Ожидаемый результат
Все BGP-сессии установлены (Established)
Все loopback-адреса доступны между узлами
Полная IP-связность внутри fabric
Underlay сеть готова для дальнейшего развертывания overlay (EVPN/VXLAN)
